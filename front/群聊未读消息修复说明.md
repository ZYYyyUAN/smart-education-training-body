# 群聊未读消息修复说明

## 问题描述111

原始问题：当点击群聊时所有消息显示为已读，但离开后收到新消息时，未读消息计数不准确。

具体表现为：
- 在同一个群聊中时，仍然显示未读消息提示
- 离开群聊后，新消息的未读计数不准确
- 重新进入群聊时，未读状态混乱

## 修复内容

### 1. 改进群聊切换逻辑

**文件：** `xiaozhi-front/src/views/teacher_chat/TeacherGroupChat.vue`

**修改内容：**
- 添加重复点击检查：如果点击的是当前已激活的群聊，不做任何操作
- 离开群聊时更新最后查看时间：确保时间戳准确记录
- 进入新群聊时清零未读计数：立即清除未读状态

```javascript
const switchGroup = (g) => {
  // 如果切换到的是当前已激活的群聊，不做任何操作
  if (activeCourse.value && activeCourse.value.courseId === g.courseId) {
    console.log(`已经在群聊 ${g.courseName} 中，无需切换`)
    return
  }
  
  // 如果之前有活跃的群聊，更新其最后查看时间
  if (activeCourse.value) {
    lastSeenMap.value[activeCourse.value.courseId] = Date.now()
    saveLastSeen()
    console.log(`离开群聊 ${activeCourse.value.courseName}, 更新时间: ${new Date().toLocaleString()}`)
  }
  
  activeCourse.value = g
  // 进入新会话即清零未读，并更新最近查看时间
  unreadCount.value[g.courseId] = 0
  lastSeenMap.value[g.courseId] = Date.now()
  saveLastSeen()
}
```

### 3. 智能未读计数管理

**文件：** `xiaozhi-front/src/views/teacher_chat/TeacherGroupChat.vue`

**修改内容：**
- 全局消息监听器：只有在离开群聊后才增加未读计数
- 轮询时强制清零：当前活跃群聊的未读计数始终为0
- 错误处理：出错时保持上次有效计数，不重置为0

```javascript
// 监听全局消息事件
window.addEventListener('newChatMessage', (event) => {
  const { courseId, message } = event.detail
  if (courseId && message) {
    // 只有在离开群聊后才增加未读计数
    if (!activeCourse.value || activeCourse.value.courseId !== courseId) {
      if (unreadCount.value[courseId] === undefined) {
        unreadCount.value[courseId] = 0
      }
      unreadCount.value[courseId]++
      console.log(`收到新消息，群聊 ${courseId} 未读消息: ${unreadCount.value[courseId]} (当前不在该群聊中)`)
    } else {
      // 当前正在该群聊中，不增加未读计数
      console.log(`收到新消息，群聊 ${courseId} 但当前正在该群聊中，不增加未读计数`)
    }
  }
})
```

### 4. 轮询时强制清零当前群聊

```javascript
// 如果当前正在该群聊中，强制清零未读计数
if (activeCourse.value && activeCourse.value.courseId === g.courseId) {
  unreadCount.value[g.courseId] = 0
  console.log(`当前正在群聊 ${g.courseName} 中，强制清零未读计数`)
}
```

## 技术细节

### 时间戳处理
- 支持数组格式：`[year, month, day, hour, minute, second]`
- 支持字符串格式：`"2024-01-01 14:30:00"`
- 自动处理时区转换

### 事件机制
- 使用 `CustomEvent` 进行组件间通信
- 全局事件监听，确保实时性
- 自动清理监听器，防止内存泄漏

### 状态管理
- 本地存储最后查看时间
- 轮询更新未读计数
- 强制清零当前活跃群聊

## 注意事项

1. **浏览器兼容性**：需要支持 `CustomEvent` 的现代浏览器
2. **内存管理**：组件卸载时自动清理事件监听器
3. **错误处理**：网络错误时保持上次有效计数
4. **性能优化**：轮询间隔为15秒，平衡实时性和性能
